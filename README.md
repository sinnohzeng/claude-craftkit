# 怎样写作 — Claude 写作润色审稿技能

> **"好文稿好文章无疑是写出来的，但更重要的是改出来的。"** — 任仲然《怎样写作》

一个基于《怎样写作》(任仲然)系统化写作方法论的 Claude Code 技能插件。帮助你审查、优化、润色各类文稿——从公文到讲话稿，从调研报告到自媒体文章。

---

## 这是什么？

**writing-polish** 是一个 [Claude Code](https://docs.anthropic.com/en/docs/claude-code) 技能（Skill），它将一本凝聚了数十年党政机关写作经验的著作——《怎样写作》——转化为 AI 可执行的审稿方法论。

当你说"帮我润色这篇文章"或"review my writing"时，Claude 会自动激活这个技能，按照**先大后小**的专业审稿流程，从立意主题到字词标点，系统化地审查和改进你的文稿。

### 核心理念

| 原则 | 含义 |
|------|------|
| **热写稿，冷改稿** | 写的时候趁热打铁，改的时候要晾一晾冷一冷 |
| **先大后小** | 先改立意主题和结构布局，再改段落语句，最后改字词标点 |
| **求精、求准** | 精：主题厚实深邃，文辞精彩朴实。准：事实准确，判断精准 |
| **以意役法** | 意（主题）是元帅，技巧和方法都要服从主题 |

## 特色功能

### 三阶段专业审稿流程

```
第一步：通读与识别
├── 完整阅读全文，不急于动笔
├── 识别文体类型（7种文体专属标准）
└── 判断修改深度（轻度润色 / 深度修改 / 结构重建）

第二步：结构性审查（大改）
├── 立意与主题 — 主题是否清晰统领全文？
├── 内容与观点 — 事实是否真实？论据是否充分？
├── 结构设计 — "四梁八柱"是否稳妥？
└── 逻辑延展 — 逻辑主线是否贯穿始终？

第三步：细节打磨（小改）
├── 语言 — 朴实精准？有无生造词汇？
├── 表达方式 — 叙述/议论/说明是否得当？
└── 字斟句酌 — 标点、用词、术语一致性
```

### 7 种文体专属审查标准

| 文体 | 核心审查维度 |
|------|-------------|
| **规范性公文** | 政治性/政策性、规范性/权威性、指导性/可操作性 |
| **领导讲话稿** | "三个吃透"、实度/深度/高度/新鲜度/气势 |
| **调研报告** | 真/实/深/高/新/活 |
| **述职报告** | 以实取胜、以数取胜、以事和绩取胜 |
| **汇报稿/发言稿** | 聚焦、建议性、目的性 |
| **随笔/杂文** | 真情智善理美、有料有趣有度 |
| **自媒体写作** | 短快真实新、精练标题、精彩开篇 |

### 何其芳 12 项修改清单

内置基于何其芳修改文章经验的系统化审校清单，覆盖从"抽象笼统"到"逗号到底"的 12 类常见写作问题，每项都有可操作的检查方法和修改建议。

### 渐进式加载

技能采用三层渐进式加载架构，节省 AI 上下文窗口：

- **第一层**（~100 词）: 技能名称 + 触发描述 — 始终加载
- **第二层**（~180 行）: SKILL.md 核心工作流 — 技能触发时加载
- **第三层**（~780 行）: 文体指南 / 审校清单 / 逻辑结构 / DOCX 编辑 / 公文排版 — 按需加载

### 多格式支持

- **Markdown**: 直接读取分析
- **DOCX**:
  - 读取：通过 pandoc 自动转换后分析
  - 编辑回写：使用 `docx-editor` 库，默认启用 Track Changes 修订模式
  - 默认修订作者："任仲然"（可指定其他名称）
- **纯文本**: 直接粘贴即可

## 安装

### 方式一：Plugin 安装（推荐）

```bash
claude /plugin install https://github.com/sinnohzeng/zen-yang-xie-zuo
```

### 方式二：手动复制

将 `skills/writing-polish/` 文件夹复制到你的 `~/.claude/skills/` 目录：

```bash
git clone https://github.com/sinnohzeng/zen-yang-xie-zuo.git
cp -r zen-yang-xie-zuo/skills/writing-polish ~/.claude/skills/
```

### 方式三：本地开发模式

```bash
git clone https://github.com/sinnohzeng/zen-yang-xie-zuo.git
claude --plugin-dir ./zen-yang-xie-zuo
```

### 前置依赖（可选）

DOCX 读取：

```bash
brew install pandoc    # macOS
apt install pandoc     # Ubuntu/Debian
```

DOCX 编辑回写（仅在需要修改 DOCX 文件并回写时）：

```bash
pip install docx-editor python-docx
```

## 使用方法

安装后，在 Claude Code 对话中使用以下任意方式触发：

### 中文触发

```
帮我润色这篇文章
请审一下这个稿子
帮我改改这篇讲话稿
优化一下这篇调研报告
这篇公文帮我看看有什么问题
```

### English Triggers

```
Polish this article for me
Review my writing
Improve this draft
Check my writing quality
```

### 指定文件

```
帮我润色 /path/to/article.md
审稿 /path/to/speech.docx
```

### 指定修改深度

```
轻度润色一下语言就好         → 侧重语言和表达
帮我全面审查这篇文稿         → 结构到细节全面审查
这篇写得不太好，帮我重新组织  → 可能涉及结构重建
```

## 方法论来源

本技能的全部方法论提炼自：

**《怎样写作》** — 任仲然 著，党建读物出版社，2019年

任仲然长期在党政机关工作，既有从事具体工作的实践，又有担任高级领导职务的经历。本书从"写作其实并不难"入手，系统讲述了写作的思维逻辑、立意构思、材料结构、语言表达，以及规范性公文、领导讲话稿、调研报告、述职报告、汇报发言稿、随笔杂文和自媒体写作的具体方法，并专门介绍了修改文稿文章的经验。

本技能从全书 12 讲中系统提取了：
- **5 种思维方式**（系统工程/唯物辩证/形象想象/创意创新/逻辑）
- **3 层审稿框架**（立意 → 结构 → 细节）
- **7 种文体专属标准**
- **12 项修改清单**（何其芳经验）
- **10 类常见问题速查表**
